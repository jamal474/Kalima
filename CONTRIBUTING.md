# Contributing to Kalima

First off, thank you for considering contributing to Kalima! 

This project is built on the philosophy that anyone should be able to use, modify, and improve it however they see fit. We welcome contributions of all kinds—whether you're fixing a bug, adding a new AI integration, tweaking the Spaced Repetition System (SRS) algorithm, or just improving documentation.

## How Can I Contribute?

### 1. Reporting Bugs
If you find a bug, please open an issue using our **Bug Report** template. Include as much detail as possible, such as your iOS version, simulator vs physical device, and steps to reproduce.

### 2. Suggesting Enhancements
Have an idea for a new feature? We'd love to hear it! Open an issue using the **Feature Request** template. Since this is an open and highly permissive project, you are highly encouraged to fork the repository and build the feature yourself!

### 3. Submitting Pull Requests
We embrace a flexible development model. If you want to contribute code:
1. **Fork** the repository.
2. **Create a branch** for your feature or bug fix (`git checkout -b feature/amazing-feature`).
3. **Commit** your changes (`git commit -m 'feat: add amazing feature'`).
4. **Push** to the branch (`git push origin feature/amazing-feature`).
5. **Open a Pull Request** using the provided PR template.

### Development Setup
Please refer to the `README.md` for instructions on setting up your local environment (e.g., configuring `GoogleService-Info.plist` and `Secrets.plist`). 

**CRITICAL RULE**: Ensure that you **never** commit your personal AI API keys or Firebase credentials!

## Project Philosophy
Do whatever you like with this code. If you find a better way to implement the SRS math, want to switch out the AI models, or want to completely redesign the UI, go for it! We just ask that you follow the Code of Conduct to keep our collaborative space positive.
